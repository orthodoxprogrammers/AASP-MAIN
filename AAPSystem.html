Key Relationships

roles→users1 : Many — each user has one role (MSWDO or Barangay)
clients→beneficiaries1 : Many — a client may be linked to a beneficiary
beneficiaries→general_intake1 : Many — one beneficiary can have multiple intake records
general_intake→financial_assistance1 : 1 — each intake has one FA record
general_intake→material_assistance1 : 1 — each intake has one MA record
general_intake→psychosocial_support1 : 1 — each intake has one PS record
beneficiaries→family_composition1 : Many — multiple family members per beneficiary
beneficiaries→beneficiary_sectors1 : Many — a beneficiary can belong to multiple sectors
beneficiaries→beneficiary_subcategories1 : Many — multiple subcategory tags per beneficiary
beneficiaries↔programsMany : Many — via beneficiary_programs junction table
beneficiaries→dswd_assistance1 : Many — history of DSWD assistance received
beneficiaries→relief_distribution1 : Many — multiple relief distributions per beneficiary

Barangay User
Limited to their assigned barangay only.
Access Control: Barangay users are restricted by their barangay column in the users table. All queries in barangay pages include a WHERE barangay = $_SESSION['barangay'] filter to enforce data isolation per barangay.
